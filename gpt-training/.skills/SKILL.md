# 技術勉強会スライド作成ガイド

## 概要

このドキュメントは、Marp 形式で技術勉強会のスライドを作成する際のベストプラクティスとプロンプトテンプレートをまとめたものです。

### 対象

- 技術勉強会・社内研修のスライド作成
- プログラミング言語、フレームワーク、設計などの技術テーマ
- 発表時間：15 分〜60 分程度

### 成果物

1. Marp スライド（slides.md）
2. 発表メモ（presentation-notes.md）
3. 画像配置用ディレクトリ（images/）

---

## Claude へのプロンプトテンプレート

### 基本プロンプト

```
Marpでスライドを作成します。
添付したマークダウンが作成してほしいスライドの概要です。

* 内容の粒度は適宜変更してもらって構いません。わかりやすく作成してください
* []内の記述は作成指示です
* 画像についてはこちらでダウンロードして適切なものを挿入するので、ディレクトリ構造について配置場所について私に指示してください
* 簡潔、かつデザイン性のあるスライドを作成してください

【以下の情報を必ず含める】
1. プレゼンテーション全体のタイトル: [タイトルを記載]
2. 発表者名や日付: 空欄にしておく
3. スライドのアスペクト比: 16:9
4. カラーテーマやデザインの方向性: お任せ
5. 対象者: [社内エンジニア/新卒研修受講者/など]
6. スライド枚数: [概要に記載した枚数を厳守]

わからないことがあれば、作業を中断し、私に質問してください
```

### スライド概要の書き方

スライド概要は以下の形式で記載する：

```markdown
1. 表紙
2. スライドタイトル
   - 説明や内容
   - 複数の要素がある場合は箇条書き
   - [画像の指示]などの特別な指示
3. 次のスライドタイトル
   ...
```

**注意点：**

- 番号付きリストで各スライドを表現
- インデントで詳細を記述
- `[]`で特別な指示（画像、図、強調など）を記載
- スライド枚数は必ず守る

## スタイル定義のベストプラクティス

### 推奨フォントサイズ（16:9、見切れ防止）

```yaml
section:
  padding: 50px 70px # 上下左右の余白

h1: 2.0〜2.2rem # スライドタイトル
h2: 1.7〜1.8rem # セクション見出し
h3: 1.2〜1.3rem # サブセクション見出し
h4: 1.0〜1.1rem # 項目見出し

p: 1.0〜1.1rem # 本文
ul/li: 1.0〜1.1rem # 箇条書き
```

### 余白とマージン

```yaml
h1/h2/h3:
  margin-bottom: 0.5〜1rem

li:
  margin-bottom: 0.3〜0.4rem

.highlight:
  padding: 0.7〜0.9rem
  margin: 0.7〜0.8rem 0
```

### カラーパレット例

**モダン・テクノロジー系（青紫）**

```css
primary: #2563eb (青)
secondary: #764ba2 (紫)
accent: #f59e0b (オレンジ)
highlight-bg: #fef3c7 (薄い黄色)
```

**シンプル・ビジネス系（グレー）**

```css
primary: #1f2937 (ダークグレー)
secondary: #6b7280 (ミドルグレー)
accent: #10b981 (緑)
highlight-bg: #e0f2fe (薄い青)
```

## レイアウトパターン

### 2 カラムレイアウト

```css
.two-columns {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 1.3〜1.5rem;
}
```

**使用例：**

- Before/After 比較
- メリット/デメリット
- コード例と説明の並列表示

### グリッドレイアウト（サービス紹介など）

```css
.service-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* 3列 */
  gap: 0.8〜1rem;
}
```

**調整のポイント：**

- 要素が多い場合：画像サイズ、padding、font-size を小さく
- 見切れる場合：gap、margin-top を小さく

### ハイライトボックス

```css
.highlight {
  background-color: #fef3c7;
  padding: 0.7rem 0.9rem;
  border-left: 4px solid #f59e0b;
  margin: 0.7rem 0;
}
```

**使用場面：**

- 重要なポイント
- まとめ
- 注意事項

## 見切れ対策チェックリスト

### よくある見切れのパターンと対策

1. **グリッドレイアウトの下部が見切れる**

   - `gap`を小さく（1rem → 0.8rem）
   - `padding`を小さく（1rem → 0.5rem）
   - 画像サイズを小さく（100px → 70px）
   - フォントサイズを小さく

2. **2 カラムレイアウトで片側が飛び出す**

   - 見出しを 2 カラムの外に配置
   - 各カラムの内容量を均等に

3. **下部の余白が不足**

   - `margin-bottom`を追加（1〜2rem）
   - 特にスライド最後の要素に注意

4. **タイトルページの文字色が見づらい**
   - グラデーション背景の場合、白文字を使用
   - `section.title strong { color: white; }`を追加

### 基本プロンプト

```

---
```

## ワークフロー

### 1. 準備フェーズ

1. **テーマと対象者を明確化**

   - 何を伝えたいか
   - 誰に伝えるか
   - どのくらいの時間か

2. **スライド概要を作成**
   - 番号付きリストで構造化
   - 各スライドの内容を簡潔に記述
   - 必要な画像や図を`[]`で指示

### 2. スライド作成フェーズ

1. **Claude にプロンプトを送信**

   - 基本プロンプト + スライド概要
   - 必須情報を含める

2. **初回出力を確認**

   - 見切れがないか
   - レイアウトは適切か
   - デザインは好みか

3. **修正指示**
   ```
   以下の点を修正してください：
   - スライドXの下部が見切れている
   - タイトルの文字が見づらい
   - グリッドの要素を小さくして
   ```

### 3. 画像準備フェーズ

1. **画像リストの確認**

   - Claude が提示したディレクトリ構造を確認

2. **画像の収集・作成**

   - ロゴ: 公式サイトからダウンロード
   - スクリーンショット: 実際の画面をキャプチャ
   - 図解: draw.io、Figma などで作成

3. **画像の配置**
   - 指示されたディレクトリに配置
   - ファイル名を厳守

### 4. 発表メモ作成フェーズ

1. **発表メモを生成**

   - Claude に発表メモ作成を依頼

2. **メモの追記**
   - 喋る内容を具体的に追記
   - タイミングや注意点を記入

### 5. 最終確認フェーズ

1. **Marp でプレビュー**

   ```bash
   marp slides.md --preview
   ```

2. **チェック項目**

   - [ ] 全スライドで見切れがない
   - [ ] 画像が正しく表示される
   - [ ] フォントサイズが適切
   - [ ] 色のコントラストが十分
   - [ ] タイポがない

3. **出力形式に変換**

   ```bash
   # PDF出力
   marp slides.md --pdf -o presentation.pdf

   # PPTX出力（Google Slidesへのアップロード用）
   marp slides.md --pptx -o presentation.pptx
   ```

## トラブルシューティング

### Q: スライドが見切れてしまう

**A: 以下の順に調整**

1. フォントサイズを小さく（-0.1〜0.2rem）
2. padding/margin を小さく
3. gap を小さく
4. 画像サイズを小さく
5. 内容を簡潔に（文章を削る）

### Q: 2 カラムレイアウトで片側が飛び出す

**A: 構造を見直す**

```markdown
悪い例：

<div class="two-columns">
  <div>
    #### 見出し
    内容
  </div>
  <div>
    内容
  </div>
</div>

良い例：

#### 見出し

<div class="two-columns">
  <div>内容</div>
  <div>内容</div>
</div>
```

### Q: タイトルページの文字が見づらい

**A: スタイルに以下を追加**

```css
section.title strong {
  color: white;
}
```

### Q: 画像が表示されない

**A: チェック項目**

- ファイルパスが正しいか（`images/`から始まる）
- ファイル名が一致しているか（大文字小文字含む）
- 画像ファイルが実際に存在するか
