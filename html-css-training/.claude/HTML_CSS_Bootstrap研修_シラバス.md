# HTML/CSS/Bootstrap 内定者研修 シラバス

## 研修概要

### 基本情報

- **研修名**: HTML/CSS/Bootstrap 入門 - すぐに作れるWeb制作研修
- **対象者**: 新卒内定者 約15名
- **実施形態**: オンライン（Zoomブレイクアウトルーム機能使用）
- **実施時間**: 13:00-17:00（実質4時間、17:00終了厳守）
- **講師**: 1名

### 対象者の特徴

- HTML/CSS完全未経験者がメイン（約70%）
- 情報学部出身者が約30%
- PC操作には慣れているが、Web制作は未経験
- 一部経験者はHTMLを少し触ったことがある程度

### 研修の目的

1. **「細かいことを知らなくても、雑な知識でそれっぽいWebサイトが作れる」を体験する**
2. HTMLの本質（文章構造のマークアップ）を理解する
3. Bootstrapを使って実践的なWebサイトを短時間で制作できるようになる
4. **AI（ChatGPT等）を活用してWeb制作を効率化する方法を学ぶ**
5. 学校教育との差別化：基礎から積み上げる「地味で退屈」ではなく、「すぐに実践できる楽しさ」を体験

### 研修の核心コンセプト

> **「概念を知らないとAIに聞けない。でも細かい文法は覚えなくていい」**

- ❌ 細かい文法やプロパティを全部覚える必要はない
- ✅ **概念は必要**：
  - HTMLとは何か（文章構造のマークアップ）
  - CSSとは何か（見た目の調整）
  - Bootstrapとは何か（CSSフレームワーク）
- **理由**：概念を知らないとAIに適切な質問ができず、AIの出力も理解・改善できない

## 研修の特徴

### 教育手法

1. **座学 → 体験のサイクル**
   - 座学で概念を理解 → 体験でアウトプット
   - 最後の演習（90分）で総合的に実践

2. **AI活用を積極的に推奨**
   - 制作中のAI使用を推奨
   - AIへの質問例をスライドで提示
   - 「わからなければすぐAIに聞く」のスタンス
   - **座学の目的：概念を理解してAIに適切に質問できるようにする**

3. **チームワークを基本**
   - 全てのワークはチーム単位（5-6人）
   - サボり防止、孤立防止
   - 相互学習の促進

4. **CSSは最小限、Bootstrapに注力**
   - CSSの基礎概念のみ教える（デザインに深入りしない）
   - Bootstrapで効率的にWeb制作
   - 公式ドキュメントのコピペ方法を教える

5. **実践重視**
   - Bootstrap演習に90分確保
   - 発表に60分確保
   - 「仕組みを理解するより、まず動かす」

### 時間管理

- 17:00終了厳守（次の用事がある学生への配慮）
- 各セクションで2-3分早めに終わることを意識
- 座学は連続35分以内（集中力維持）
- 演習と発表に多くの時間を割く

## 未経験者への配慮

### つまずきポイントと対策

#### つまずきポイント1: VSCode + Live Serverの使い方がわからない

**対策**:
- 開始前に画面共有でデモを見せる
- セットアップ手順書を配布
- 動かない場合は個別サポート

#### つまずきポイント2: HTMLタグの閉じ忘れ

**対策**:
- 「開いたら必ず閉じる」を強調
- エラーが出たらまずタグの閉じ忘れを確認
- VSCodeの自動補完を活用

#### つまずきポイント3: Bootstrapクラス名のスペルミス

**対策**:
- チートシートを配布
- 公式ドキュメントのコピペを推奨
- AIに聞く方法を教える

#### つまずきポイント4: 画像が表示されない

**対策**:
- 画像素材サイト（Unsplash、Pixabay）のURLを直接使う方法を教える
- テンプレートにサンプル画像URLを記載

#### つまずきポイント5: レスポンシブ対応ができない

**対策**:
- Bootstrapのグリッドを使えば自動的にレスポンシブになることを強調
- デモで画面サイズを変えて見せる

### サポート体制

- **ブレイクアウトルーム巡回**：講師が7部屋を巡回
- **チーム内サポート**：5-6人チームなので誰かが詳しい可能性が高い
- **チャットで質問**：メインルームのチャットで質問可能
- **リソース提供**：チートシート、画像素材リスト、セットアップ手順書

## 補足事項

### 進行上の注意点

- 未経験者が多いため、専門用語は避け、平易な言葉で説明する
- 「概念を知らないとAIに聞けない」を繰り返し強調
- ブレイクアウトルームでは講師が巡回し、困っているチームをサポートする
- 時間に余裕を持たせ、遅延リスクを吸収できるバッファを含める

### 柔軟な対応

- 未経験者が多く遅延しそうな場合、座学②の一部を簡略化
- 演習時間が足りない場合、必須要件のみに絞る
- 発表時間が足りない場合、各チーム6分に短縮

### AI活用の推奨

- **制作中のAI使用を積極的に推奨**
- AIへの質問例をスライドで提示
- 「わからなければすぐAIに聞く」のスタンスを強調
- 発表時に「AIをどう活用したか」を共有（任意）

### 研修後のフォローアップ

- 研修資料・成果物のURLを共有
- さらに学びたい人へのリソース紹介（Progate、ドットインストール、MDN Web Docs）
- 質問があればいつでも連絡可能
